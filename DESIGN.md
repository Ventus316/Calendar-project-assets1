# 難哄・日曆 (Nan Hong Calendar) - 完整設計規範文件

> **專案概述：** 本專案以電視劇《難哄》為核心主題，將劇集情感轉化為沈浸式網頁互動體驗。透過 32 天的倒數機制與三階段狀態機邏輯，重現主角桑延與溫以凡從重逢到圓滿的情感軌跡。

---

## 1. 設計理念 (Design Philosophy)

- **核心隱喻：** 將 32 集劇情濃縮為 12/01 至 01/01 的每日心動語錄，強調時間積累的儀式感。
- **情感敘事維度：**
  - **初見 (Step 1)：** 播放轉場影片後僅顯示標題，象徵初期的隔閡與試探。
  - **交心 (Step 2)：** 透過互動解鎖核心語錄與明日預告，象徵情感的共鳴與升溫。
  - **深情 (Easter Egg)：** 隱藏的彩蛋影片，象徵內心深處不為人知的守護與救贖。

---

## 2. 視覺系統 (Visual System)

### 2.1 色彩規範 (Color Palette)

| 色彩名稱   | Hex Code  | 應用場景            | 設計意涵                           |
| :-------- | :-------- | :------------------ | :-------------------------------- |
| **霜降黑** | `#1a1a1a` | 全局背景、卡片底色   | 象徵桑延深沉的守護與初期的距離感。   |
| **烈陽金** | `#d4af37` | 核心語錄、日期、高亮 | 象徵溫暖、希望與「烈陽」般的救贖。   |
| **星塵灰** | `#b0a4a4` | 次要資訊、未解鎖日期 | 輔助視覺，不奪取焦點，傳遞經典質感。 |

### 2.2 互動視覺特效

- **水波紋特效 (Ripple Effect)：** 點擊螢幕時產生具備毛玻璃感（`backdrop-filter`）的圓環擴散，模擬心動的漣漪。
- **動態日曆：** `active-day` 具備呼吸燈動畫（Pulse Animation），指引使用者當前進度。

---

## 3. 核心技術實作 (Technical Implementation)

### 3.1 三階段狀態機 (State Machine Logic)

程式透過 `currentStep` 變數精準控制卡片內容的呈現節奏：

1. **階段 0 → 1：** 點擊日曆 -> 播影片 1 -> 顯示 **日期與標題**。
2. **階段 1 → 2：** 點擊背景遮罩 -> 播影片 2 -> 解鎖 **語錄、明日預告與彩蛋按鈕**。
3. **階段 2 → 3：** 點擊彩蛋 -> 播影片 3 -> 保持完整內容顯示並更新進度。

### 3.2 響應式與縮放機制

- **Detail Scaler：** 透過 JavaScript 動態計算視窗比例，利用 `transform: scale()` 確保 900x500 的設計稿在不同設備上皆能完美適配且不溢出。
- **Grid 自適應：** 根據起始日期自動判斷日曆行數（5 列或 6 列），維持視覺平衡。

---

## 4. 輔助系統 (Support Systems)

### 4.1 開發者調試面板 (Debug Panel)

- **連點機制：** 點擊製作人資訊 5 次可開啟開發者模式。
- **自由導航：** 允許手動設定 `Day 1` 起始日期，以便測試不同年份與月份的排版。
- **進度無視：** 開發者模式下可自由點選鎖定日期，方便素材審查。

### 4.2 資源優化

- **預加載 (Preloading)：** 針對首集影片與核心視覺圖片（`bj-gq.jpg`, `heng.png`）進行預載，減少轉場黑屏。
- **本地儲存 (LocalStorage)：** 自動紀錄 `loveCalendarProgress`，確保使用者體驗的連續性。
